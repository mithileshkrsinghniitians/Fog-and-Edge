# Grafana Cloud Setup — Smart Energy Grid Dashboard

This is my step-by-step guide for getting Grafana Cloud connected to the Lambda API
and showing live sensor data. I wrote this as I was setting it up so it's pretty
detailed — future me will thank present me for this.

---

## What we're building

```
DynamoDB → Lambda (query_handler) → Grafana Cloud
                ↑
         polls every 30s
```

Grafana will call our `query_handler` Lambda Function URL every 30 seconds,
get the latest sensor readings, and display them on 5 panels.

---

## Step 1 — Sign up for Grafana Cloud (free tier)

1. Go to **https://grafana.com** and click **"Create free account"**
2. Sign up with your email (or GitHub/Google)
3. Choose a stack name — something like `smart-energy-grid`
4. Select the **EU (Ireland)** region (closest to our AWS eu-west-1 setup)
5. Click **"Start using Grafana"**

You'll land on the Grafana Cloud portal. Click **"Launch"** next to your stack
to open the actual Grafana dashboard editor.

The free tier gives you:
- 10,000 metrics/month
- 14 days data retention
- 3 active users
- Enough for this project.

---

## Step 2 — Install the Infinity Datasource Plugin

We need a plugin called **Infinity** to call our Lambda's JSON API.
The built-in Grafana datasources are for things like Prometheus and Graphite —
they don't know how to call a custom JSON endpoint. Infinity fills that gap.

1. In Grafana, go to **Home → Administration → Plugins**
   (or click the puzzle piece icon in the left sidebar)
2. Search for **"Infinity"**
3. Click on **"Infinity"** by Yesoreyeram
4. Click **"Install"** (this takes about 30 seconds on Cloud)
5. Wait for the green "Installed" badge to appear

---

## Step 3 — Configure the Datasource

Now point the Infinity datasource at our Lambda URL.

1. Go to **Home → Connections → Data sources**
2. Click **"Add new data source"**
3. Search for **"Infinity"** and select it
4. In the settings page:
   - **Name**: `Smart Energy Grid API`
   - **Base URL**: paste your Lambda Function URL here
     - It looks like: `https://abc123xyz.lambda-url.eu-west-1.on.aws`
     - Find it in AWS Console → Lambda → `query_handler` → Configuration → Function URL
   - Leave everything else as default
5. Scroll down and click **"Save & test"**
   - If it works, you'll see a green "Data source connected" message
   - If it fails, check that your Lambda Function URL has auth type set to NONE
     (AWS Console → Lambda → Function URL → Auth type → NONE)

---

## Step 4 — Import the Dashboard

The easiest way to get the 5 panels set up is to import the pre-built JSON file.

1. Go to **Home → Dashboards**
2. Click the **"New"** button (top right) → **"Import"**
3. Click **"Upload dashboard JSON file"**
4. Select `backend/dashboard/dashboard_config.json` from this project
5. On the import screen:
   - **Name**: Smart Energy Grid (or whatever you like)
   - **Datasource**: select `Smart Energy Grid API` (the one we just created)
6. Click **"Import"**

You should now see all 5 panels. They'll be empty until the Lambda has data in
DynamoDB — run the sensor simulator and fog node for a few minutes first.

---

## Step 5 — Configure the Lambda URL Variable

The dashboard uses a template variable `lambda_url` so you don't have to edit
every single panel if the URL changes.

1. Open the imported dashboard
2. Click the **gear icon** (Dashboard settings) at the top right
3. Go to **Variables**
4. Click on **`lambda_url`**
5. Set the **Default value** to your actual Lambda Function URL
6. Click **"Update"** then **"Save dashboard"**

Now all 5 panels will automatically use that URL.

---

## Step 6 — The 5 Panels (what each one shows)

After import, you should see these panels:

**Panel 1 — Solar Output** (top left, green lines)
- Shows kW generated by solar panels over time
- Three lines, one per home — they'll diverge during cloudy periods
- Expected range: 0–4.5 kW. Flatlines to 0 at night.

**Panel 2 — Grid Consumption** (top right, red lines)
- kW being drawn from the electricity grid
- Spikes in morning (7–9am) and evening (6–9pm) are normal
- A sudden spike with EV_CHARGING mode = car plugged in

**Panel 3 — Battery Level** (middle left, blue lines)
- State of charge as a percentage (0–100%)
- Should rise during daytime (solar charging) and fall overnight
- BATTERY_LOW alert triggers when this drops below 15%

**Panel 4 — EV Charger** (middle right, yellow bars)
- Either 0 kW (idle) or 7.4 kW (charging session active)
- Bar chart makes the on/off nature clearer than a line chart

**Panel 5 — Temperature** (bottom, full width, orange lines)
- Simulated indoor ambient temperature in °C
- Drifts slowly toward the time-based target (17°C night, 21°C morning/evening)

## Step 7 — Set Auto-Refresh to 30 Seconds

Our fog node sends new data every 30 seconds, so Grafana should refresh at the same rate.

1. In the dashboard, find the **refresh interval** dropdown at the top right
   (it might say "Off" or "5m" by default)
2. Click it and select **"30s"**
3. You can also click the dashboard settings gear → set the **"Auto refresh"** default
   to `30s` so it persists for everyone who opens the dashboard

---

## Step 8 — Create a Battery Low Alert

This sets up a Grafana alert that fires when any home's battery drops below 15%.

1. Open the **Battery Level** panel
2. Click the panel title → **"Edit"**
3. Go to the **"Alert"** tab (bell icon)
4. Click **"Create alert rule from this panel"**
5. Configure the alert:
   - **Alert name**: `Battery Low Warning`
   - **Evaluate every**: `30s`
   - **Conditions**: `WHEN last() OF query IS BELOW 15`
6. In the **"Notifications"** section:
   - Add a contact point (email or Slack — set this up in Alerting → Contact points first)
   - Set a message like: `Battery below 15% — check home energy system`
7. Click **"Save rule and exit"**

To set up email notifications:
- Go to **Alerting → Contact points → Add contact point**
- Type: Email, add your email address
- Test it with the "Test" button

## Troubleshooting

**Panels show "No data"**
- Make sure the fog node and sensors are running
- Check the Lambda is getting invoked: AWS Console → Lambda → Monitor → View CloudWatch logs
- Try calling the Lambda URL directly in your browser: `{lambda_url}?hours=1`

**"Datasource not found" error**
- Re-link the datasource: Dashboard settings → Variables → lambda_url → update the URL

**Data looks stale**
- Check the auto-refresh is set to 30s
- Confirm the fog node's processing loop is running (you should see [FOG] logs every 30s)

**"Connection refused" from Infinity datasource**
- Lambda Function URL must have Auth type = NONE
- Check the URL hasn't expired (Lambda URLs are permanent unless you delete them)
- Check CORS is enabled in query_handler.py — it should be, we added CORS headers
